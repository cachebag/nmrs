# Contributing

## Code of Conduct
- Maintain professionalism and respect in all communication.  
- Review your code before submitting; test it, document it, and ensure it compiles.  
- Do not submit low-effort or autogenerated code. If you absolutely must, please disclose _how_ you used AI otherwise I will close the PR.  

## Requirements

**To run or develop nmrs you need:**
- Rust (stable) via `rustup`
- GTK4 and libadwaita development libraries
- A running NetworkManager instance 
- A Wayland compositor 

On Arch Linux:
```bash
sudo pacman -S gtk4 libadwaita base-devel
````

On Debian/Ubuntu:

```bash
sudo apt install pkg-config libglib2.0-dev libgirepository1.0-dev \
libgdk-pixbuf2.0-dev libpango1.0-dev libcairo2-dev \
libgtk-4-dev libadwaita-1-dev
```

## Project Structure

```bash 
nmrs-core/   – D-Bus backend. Handles connections, access points, and NetworkManager IPC.
nmrs-ui/     – GTK4 frontend. Displays network state, handles user interaction.
```
#### References: 
- https://networkmanager.dev/docs/api/latest/
- https://gtk-rs.org/gtk4-rs/stable/latest/docs/gtk4/

## Guidelines

* Keep PRs focused and logically scoped.
* Avoid adding dependencies without discussion.
* Respect the existing module structure and conventions.
* CI must pass before merge.

## Issues

* Use the provided templates when opening issues.
* Include system details (`uname -a`, compositor, distro).
* Screenshots are encouraged for UI reports.
* Tag with appropriate labels (`bug`, `feature`, `discussion`).
